Preserving Computational Objects - Homer and Darwin

 

Modern cloud execution models such as “serverless computing” (FaaS) create stateless API ecosystems that allow the code to move to the data, rather than the more conventional have the data travel to where the application is deployed. The latter is much harder to scale and is nearly impossible to govern transparently. This presentation will illustrate this mechanism with applications to population studies, from genomics to pathology. Let's have a look at how computational ecosystems in the Web Era approach using popular APIs for biomedical applications behave...

    5 points for discussion towards learning health systems

    1. How do you train a neural network: best video ever. Yes, we've been teaching this for 2 decades ...

     

    2. Grad students now do it as part of their thesis, but then don't stay in the extramural funding model, they prefer to stay close to the data.

     

    3. It doesn't require specialized hardware to start - neural computing playgrounds can now be injected in the browser ...

     

    4. ... access to the cloud (NIH STRIDES helps do it serverless!!!) scales it up.

     

    5. Looking further afield - federated learning is enabling a global Web+Cloud machine, and even that is not new. Note how a) the most popular neural computing library has a browser implementation, and so do others, which b) can engage mobile, user-governed, computing.
